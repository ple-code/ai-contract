## 二、业务现状（为什么痛点集中在"比对"）

1. **陪标是刚需**：同一项目经常要**同时写多份方案**（自家 + 陪标方），且**格式、模板、内容、风格必须完全区分开**——一旦被招标方查重系统判为重复 → 直接**废标 / 串标**，后果是 **1–3 年内不能再投标**。
2. **现状靠人工 3 人交叉核验**：标书页数动辄**几百到上千页**，肉眼比对"特别累"，而**人工最容易出错**（一分心就漏）。
3. **现有工具不及格**：
   - **WPS 比对**：只能在"相同位置/同级标题"下逐字比对，**跨位置重复抓不到**；且要先传文件、选两份、操作繁琐。
   - **投标大王**（董大/董林那边的网页工具，公网 IP，非公司产品，仅给试用）：生成的标书**格式很乱**——一级/二级标题错位、序号跳号（9.1.1）、附件/授权书/身份证位置错乱。
4. **行业有付费方案但贵**：业内有一款 **"比对网"软件，4 万块永久授权、单机版 + 加密狗、只绑一台设备**——证明**技术可行**，但大家都是各家自用、价格不低。

## 三、核心需求（业务方本场明确收敛为两点）

> 王亚楠原话：**"我觉得就两点就好了——第一是招标公告加政策的爬虫；第二就是这个（AI 智能比对），对就好了。"**

### 📌 模块一（P0）：AI 智能比对 —— 孟总最大需求

**目标**：写完多份陪标方案后，**自动比对**是否会被招标方查重系统判定为重复，规避串标风险。

**比对维度（业务方提出的规则）**：

| 维度 | 具体规则 |
|---|---|
| **文字查重** | 连续相同字数有阈值（举例：**连续 5 个字一样**就告警）；分词后不能整段重复 |
| **图片查重** | 图片不能一样 |
| **语言风格** | 投标惯用语（如"该系统支持……"）每份都必须区分，不能套同一模板 |
| **格式风格** | 整体排版风格不能太相似（这条**主观**，AI 方提示需先给客观规则） |

**特殊处理**：
- **剔除招标文件内容**：方案里会**复制招标文件原文**（这部分重复是正常的），比对前应能**剔除招标文件**，只看自家写出的方案之间是否撞车。
- **多份比对**：一次上传 3–4 份方案相互比对（不是只两份）。
- **可生成多份差异化方案（高阶，延后）**：根据一份需求**一次生成 3 份完全不同的描述**——但 AI 方提示需先建业务知识库；业务方担心"AI 学过之后下次会不会串"，AI 方确认知识库可定制、生成内容**不回流**到知识库。

**比对的输出**：不只是"一样/不一样"，要**标出哪些是相似的**（关键词/图片），方便回到文档里改。

**交互形式**：网页即可（业务方明确："你们先做一个网址吧"）。

### 📌 模块二（P1）：招标公告 + 政策爬虫

**目标**：定期自动抓取并**推送招标公告/招投标政策到邮箱**，不错过采购机会。

- **来源**：政府采购网（国家 / 各省 / 各市县）。
- **形式**：关键词定向爬取；按周/月推送到邮箱（业务方提到侯总此前找"永恒"做过类似工作）。
- **登录站支持**：部分行业性官网需账密登录才能看到 → **可支持**，账密存内网服务器，安全由运维侧保障。
- **不做的**：登录站如果技术上不好做，优先做免登录的公告/政策类。

### 📌 模块三（P2，业务方明确"没必要"）：标书起草

**业务方态度**：**不推荐做**。

- 理由：试过"投标大王"等工具，**格式生成乱**、生成的内容也只是骨架、真正的内容还得自己写——"**手搓会比系统生成快**"。
- AI 团队定位：仍会做一版**应付演示/比赛**用，不会投入很深；同时按业务方要求**重点把精力放到模块一**。

## 四、招投标业务流程（AI 方梳理，业务方确认）

```
客户要做项目
   ↓
销售前期介入（敏感，会上不展开）
   ↓
[招标公告挂网]  ← 模块二爬虫在这里抓
   ↓
AI 团队只有 AI 产品现成方案；项目型需定制（售前授权写完给）
   ↓
按招标文件写投标文件  ← 模块三起草（业务方觉得没必要）
   ↓
[多份方案交叉比对、规避查重]  ← 模块一智能比对（核心痛点）
   ↓
应标 → 中标 → 给代理机构服务费（几千~上万/次）
```

**关键行业事实**：
- **代理机构极度分散**：每个市县都有，**全国几百几千家**，没有全国性大机构垄断。
- **每家代理机构的招标文件格式都不一样**，且**要求投标方响应文件格式跟它一致** → 这是"统一模板"在行业里走不通的根本原因。
- **招标形式本身也很多种**（公开/邀请/竞争性谈判等）。

## 五、技术约束与安全（反复强调）

| 约束 | 说明 |
|---|---|
| **数据极敏感** | 招投标+陪标属公司机密，老板关注自主可控；线上不能给外网模型 |
| **首选内网模型** | 用**云生办公网内的大模型**（不连外网）；模型服务器在公司三楼机房 |
| **demo 阶段** | 先**公网部署**做调试，业务方用**百度搜来的样例文件**测试，不用自家文件 |
| **上线阶段** | 找**云生运维**开模型 key + 拉**安全部门**一起对 → 才能用真实文件 |
| **离线版** | 成本极高（需在本地配高配显卡跑开源大模型），不优先考虑 |
| **账密安全** | 爬虫需要的登录账密存内网，泄露风险由运维侧兜底 |

> AI 方原话：**"如果（云生办公网模型）这个都不可以的话，那你们这个需求是很难做了。"**
