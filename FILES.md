# 项目文件说明 (FILES.md)

> 逐项说明 AI-Contract 项目中每个文件 / 目录的用途。新增、删除、改用途时同步更新本文件。

## 目录总览

```
ai-contract/
├── README.md             # 项目说明（概览 + 工作流）
├── FILES.md              # 项目文件说明（本文件，逐项清单）
├── .gitignore            # 忽略音频 / 虚拟环境 / 缓存等
├── .git/                 # 版本控制
├── raw/                  # 原始资料（录音 + 笔记）
├── analysis/             # Claude 产出（转写 + 摘要，平铺）
├── tmp/                  # 中间产物（已忽略）
└── .venv/                # Python 虚拟环境（FunASR 等，已忽略）
```

## 文件 / 目录详细说明

### 1. README.md
- **作用**：项目"门面"说明，介绍项目简介、目录结构、工作流、命名规范。
- **面向**：任何首次接触本项目的人。
- **与 FILES.md 的区别**：README 偏"概览 + 流程"；FILES 偏"逐文件清单"。

### 2. FILES.md
- **作用**：本文件，逐项描述每个文件/目录用途。

### 3. .gitignore
- **作用**：声明哪些文件不入 git。
- **当前忽略**：`.venv/`、`tmp/`、`raw/*.m4a`（及其他音频格式）、`.cache/`、`.DS_Store`、`.obsidian/workspace*` 等。
- **为什么忽略音频**：体积大（单文件 10–40 MB），且可从原来源恢复；文本/JSON/Markdown 全部入库。

### 4. raw/ （原始资料目录）
- **作用**：存放项目原始输入——调研录音 + 业务方提供的文字补充。
- **命名规范**：`YYYYMMDD-人名-场景.扩展名`（仅日期 + 描述，无时分）。
  - 录音示例：`20260626-沈泰-电话.m4a`、`20260626-李颖莹-面聊.m4a`、`20260626-王亚楠_面聊.m4a`
  - 文字示例：`20260626-李颖莹-文字.md`（业务方手写的补充说明）
- **特性**：原始输入，**录音不入 git**（被 `.gitignore` 排除），文字笔记入库。
- **占位文件**：`.gitkeep`。

### 5. analysis/ （Claude 产出目录，平铺）
- **作用**：存放 Claude 所有产出，**全部平铺、不分子目录、不分可改/不可改**。
- **文件类型**（同名前缀对应同一次调研）：
  - `*_说话人分离.md` / `*_说话人分离.txt` —— 带 说话人A/B/C… 标签的转写
  - `*_raw.json` —— FunASR 原始输出（含时间戳、spk 等）
  - `*_需求摘要.md` —— Claude 起草的结构化需求摘要
- **编辑约定**：**你想改哪个直接改**，git 兜底。详见 `analysis/README.md`。
- **占位文件**：无（已有真实文件）。

### 6. tmp/ （中间产物）
- **作用**：临时文件、调试输出。
- **特性**：已 `.gitignore`，不入库；可随时清空。

### 7. .venv/ （Python 虚拟环境）
- **作用**：本地 Python 环境，安装 FunASR、PyAV、ModelScope 等依赖。
- **特性**：已 `.gitignore`，不入库。
- **重建**：在新机器上 `python3 -m venv .venv && source .venv/bin/activate && pip install funasr modelscope av` 即可。

### 8. .obsidian/ （Obsidian 配置）
- **作用**：本项目同时作为 Obsidian 笔记库使用，此目录存 Obsidian 配置。
- **特性**：核心配置入库，`workspace*` 等本地状态忽略。

## 维护约定

1. 新增/删除文件或目录时，**必须**同步更新"目录总览"与"详细说明"。
2. 文件用途发生本质变化时，**应当**更新本说明。
3. `.gitkeep` 为占位文件，无需单独说明业务用途。
4. `.venv/`、`tmp/`、`.git/` 等基础设施目录，知道用途即可，不展开。
