# 项目文件说明 (FILES.md)

> 本文件用于逐项说明 AI-Contract 项目中每个文件 / 目录的用途，方便快速了解项目构成。
> 当新增或删除文件时，请同步更新本说明。

## 目录总览

```
ai-contract/
├── README.md             # 项目说明
├── FILES.md              # 项目文件说明（本文件）
├── .gitignore            # 忽略音频/虚拟环境/缓存等
├── raw/                  # 原始需求 / 资料文件（音频、笔记）
└── analysis/             # Claude 产出的转写 + 摘要（全部平铺，详见 analysis/README.md）
    ├── README.md         # 目录约定（git 兜底）
    ├── *_说话人分离.{md,txt}
    ├── *_raw.json
    └── *_需求摘要.md
```

## 文件 / 目录详细说明

### 1. README.md
- **类型**：Markdown 文档
- **作用**：项目的"门面"说明文件，介绍项目简介、目录结构、命名规范、使用流程等。
- **面向对象**：任何首次接触本项目的人。
- **维护建议**：项目结构或核心流程发生变化时更新。

### 2. FILES.md
- **类型**：Markdown 文档
- **作用**：逐项描述项目中每个文件 / 目录的用途（即本文件）。
- **与 README 的区别**：README 偏"概览与流程"，FILES 偏"逐文件清单"。
- **维护建议**：新增、删除、改用途的文件都要同步更新本表。

### 3. raw/ （原始需求目录）
- **类型**：目录
- **作用**：集中存放项目原始需求文档、参考资料等输入文件。
- **命名规范**：`YYYYMMDD-HHmm_原始文件名.扩展名`（时间为放入时间）。
- **特性**：原始、只读、可追溯，**不要直接修改里面的内容**。
- **占位文件**：`.gitkeep`（让空目录可被版本控制追踪，本身无业务含义）。

### 4. analysis/ （Claude 产出目录，平铺）

所有 Claude 产出的文件平铺在这里，**不分子目录、不分可改/不可改**：

- `*_说话人分离.{md,txt}` —— 转写
- `*_raw.json` —— 转写原始 JSON
- `*_需求摘要.md` —— 摘要

**你直接改任何文件**，git 兜底。Claude 重新生成前会先 `git status` 检查，未提交的改动会先问你。详见 `analysis/README.md`。

### 5. scripts/ingest.sh
- **类型**：Bash 脚本
- **作用**：把外部文件存入 `raw/`，并自动在文件名前加上"放入时间"前缀。
- **命名规则**：`YYYYMMDD-HHmm_原始文件名`，同名文件自动追加序号避免覆盖。
- **使用方式**：
  ```bash
  ./scripts/ingest.sh 需求文档.docx 参考资料.xlsx
  ```
- **依赖**：Bash 环境（macOS / Linux 默认可用）。
- **维护建议**：若命名规范调整（如时间格式变更），同步修改本脚本与 README、FILES 中的相关描述。

## 维护约定

1. 新增文件 / 目录时，**必须**在"目录总览"与"详细说明"中补齐对应条目。
2. 删除文件 / 目录时，**必须**同步移除本文件中的对应说明。
3. 文件用途发生本质变化时，**应当**更新本说明，避免与实际行为不符。
4. `.gitkeep` 为占位文件，无需单独说明其"业务用途"。
