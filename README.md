# AI-Contract 项目说明

## 项目简介

AI-Contract 是一个面向合同场景的人工智能项目，旨在通过 AI 能力辅助合同的生成、审查、分析与管理工作，提升合同处理的效率与准确性。

## 目录结构

```
ai-contract/
├── README.md             # 项目说明
├── FILES.md              # 项目文件说明（逐项描述每个文件 / 目录用途）
├── raw/                  # 原始需求/资料文件（文件名带"放入时间"前缀）
├── analysis/             # AI 分析结果（经确认、值得保留的产出）
└── scripts/
    └── ingest.sh         # 辅助脚本：把文件存入 raw/ 并自动加时间戳
```

> 详细的逐文件说明见 [FILES.md](./FILES.md)。

## 目录说明

| 目录 / 文件 | 说明 |
| --- | --- |
| `raw/` | 存放项目原始需求文档、资料文件。文件名统一为 `时间_原始文件名`，时间为"放入 raw 的时间" |
| `analysis/` | 存放 AI 分析的结果。**仅保留经确认、确实有价值的产出**，避免冗余堆积 |
| `scripts/ingest.sh` | 辅助脚本，自动给文件加时间戳前缀并存入 `raw/` |
| `README.md` | 当前项目说明文件（概览与使用流程） |
| `FILES.md` | 项目文件说明，逐项描述每个文件 / 目录的用途 |

## 文件命名规范

### raw/ 目录

格式：`YYYYMMDD-HHmm_原始文件名.扩展名`

- 时间 = 文件被放入 `raw/` 目录的时间
- 示例：`20260626-1738_合同需求文档.docx`

**推荐方式**：使用脚本自动加前缀，避免手动改名出错。

```bash
# 把一个或多个文件存入 raw/（自动加时间戳前缀）
./scripts/ingest.sh 需求文档.docx 参考资料.xlsx
```

### analysis/ 目录

- 存放 AI 分析的最终产出（用户确认要保留的内容）
- 建议命名清晰，可按主题/日期归类，例如：`20260626-合同条款风险点分析.md`

## 使用流程

1. **收集原始资料**：把需求文档通过 `./scripts/ingest.sh` 存入 `raw/`（自动加时间戳）。
2. **AI 分析**：基于 `raw/` 中的原始文件进行分析。
3. **沉淀结果**：仅将"确认有必要保留"的分析产出存入 `analysis/`，其余可丢弃。

## 备注

- 项目当前处于初始阶段，后续将根据 `raw/` 中的需求文件补充详细设计与开发计划。
- `raw/` 强调"原始、只读、可追溯"；`analysis/` 强调"精炼、有价值、可复用"。
