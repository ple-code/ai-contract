# UI 风格参考（设计基准）

| 项 | 内容 |
|---|---|
| 状态 | 基准草案，供前端实现参考 |
| 原型来源 | `daily-chat/合同比对工作台.html`（高保真静态原型） |
| 定位 | 合同审阅工作台的视觉与交互基准；后续页面在此风格上扩展 |

> 本文从原型中提炼设计语言（design tokens + 布局 + 组件），作为前端落地的统一基准。原型本身可直接在浏览器打开查看效果。

---

## 一、整体风格

**中式法务严肃风**：藏青为主色、点缀金作强调、纸白底，配衬线中文字体排合同正文，营造"正式、克制、可信赖"的法律文书气质。整体小圆角（2px）、低饱和、重留白，不花哨。

---

## 二、设计 Token

### 2.1 颜色

| 语义 | 变量 | 色值 | 用途 |
|---|---|---|---|
| 主色·藏青 | `--navy` | `#1a2740` | 顶栏、标题、主按钮 |
| 藏青·深 | `--navy-deep` | `#11192b` | 强调金上的文字底 |
| 藏青·柔 | `--navy-soft` | `#2c3b5a` | 按钮 hover |
| 正文墨色 | `--ink` | `#1c2230` | 正文 |
| 墨色·柔 | `--ink-soft` | `#4a5568` | 次要文字、说明 |
| 纸白 | `--paper` | `#fcfcfa` | 卡片/面板底 |
| 纸白·暗 | `--paper-dim` | `#f4f5f2` | 页面背景、表头底 |
| 分割线 | `--line` / `--line-strong` | `#e3e5e0` / `#c9ccc4` | 边框、分隔 |
| 删除·红 | `--del` / `--del-bg` | `#b3261e` / `#fbeae8` | diff 删除内容 |
| 新增·蓝 | `--ins` / `--ins-bg` | `#1d5d8c` / `#e7f0f6` | diff 新增内容 |
| 风险·高/中/低 | `--risk-high/mid/low` | `#b3261e` / `#b07908` / `#5a6b52` | 风险等级标识 |
| 点缀·金 | `--gold` | `#b08d4f` | 强调、AI 标记、当前阶段、激活态左条 |

### 2.2 字体

- 标题 / 合同条款正文：`Noto Serif SC`（衬线，体现文书感）。
- UI / 按钮 / 说明文字：`Noto Sans SC`（无衬线）。
- 数字（金额、账期、条款号）：`font-variant-numeric: tabular-nums` 等宽对齐。

### 2.3 其他

- 圆角：统一 `2px`（克制、偏方正）。
- 顶栏底部 `2px` 金色描边作为品牌分隔。
- 卡片用 1px 细线 + 顶部 3px 主色/金色描边区分重点区块。

## 三、布局结构

```
┌─ 顶栏 topbar（藏青底，金色下边）────────────────────────┐
│  品牌「明衡 · 合同审阅工作台」      角色流转：销售→法务→商务→定稿 │
├─ 文档信息条 docbar（纸白）──────────────────────────────┤
│  合同标题 / 编号 / 当前在谁手上     版本 badge v1→v2 · 导出报告  │
├──────────────┬──────────────────────────────────────────┤
│ 左侧变更导航   │ 主内容区 main                              │
│ sidenav 240px │  ① 关键字段变更摘要（金额/账期/违约金/交期 4 格）  │
│  · 本轮变更列表 │  ② 条款卡片列表（逐条）                       │
│  · 风险点圆点   │     - 卡头：条款名 + 修改/新增标签 + 接受/拒绝/批注 │
│  · 点击定位     │     - 双栏对照：v1 旧 | v2 新（行内 diff 红删蓝增） │
│               │     - AI 解读条（金色「AI 解读」标 + 立场化解读）    │
└──────────────┴──────────────────────────────────────────┘
```

响应式：`max-width: 900px` 时左侧导航隐藏、双栏对照变单栏、摘要 4 格变 2 格。

## 四、关键组件与 PRD 功能映射

| 原型组件 | 对应 PRD 需求 | 说明 |
|---|---|---|
| 顶栏「角色流转」(销售→法务→商务→定稿) | 流程/角色 | 体现合同流转阶段，当前阶段金色高亮 |
| 版本 badge（v1 销售提交 → v2 当前） | F5、F11 | 版本/修改记录留痕 |
| 左侧「本轮变更」导航 + 风险圆点 | F14、F3 | 按变更/风险点一键定位条款，免翻整篇 |
| 关键字段变更摘要（4 格） | F3 | 自动监控金额/账期/违约金/交期等关键字段变更 |
| 条款双栏对照 + 行内 diff（del/ins） | F3 | 文档/版本比对（篡改检测）的可视化 |
| 卡片「接受 / 拒绝 / 批注」 | F5、F2 | 修改建议一键应用/驳回 + 批注留痕 |
| 「AI 解读」条（立场化解读） | F2、F4、F6 | 初审解读，体现甲/乙/中立立场倾向，可挂法律依据 |

> 落地提示：原型中风险等级（高/中/低圆点）、立场化解读（"利于我方/略不利"）需与 F4 默认甲方立场、F2 风险识别结果联动。

## 五、交互基调

- 风险/变更**优先置顶**（摘要在前、逐条在后），减少法务通读成本。
- 操作克制：每条变更就近提供"接受/拒绝/批注"，避免跳转。
- AI 产出统一以「AI 解读 / 建议」语义呈现，呼应"初审、待复审、不直接采信"的定位（见 PRD 6.3）。
