# docs/ — 产品交付文档

本目录承载从「调研」走向「落地」的全部产出国文档。与 `analysis/`（调研中间产物）分开管理，这里放的是**面向工程交付**的正式文档。

## 目录结构

| 目录 | 用途 | 典型文件 |
|------|------|----------|
| `01-需求文档/` | 产品需求、用户故事、PRD | `PRD-v1.md`、`需求清单.md`、`用户故事.md` |
| `02-技术方案/` | 架构、技术选型、API、数据模型 | `架构设计.md`、`技术选型.md`、`API设计.md` |
| `03-研发任务拆解/` | 研发任务项 + 进度追踪 | `任务清单.md` |
| `04-测试用例/` | 测试方案、用例集、验收标准 | `测试方案.md`、`用例集.md` |
| `05-部署/` | 部署脚本、环境配置、运维手册 | `部署手册.md`、`环境配置.md` |

> 目录用 `01-` ~ `05-` 数字前缀排序，对应产品交付自然流程：**需求 → 方案 → 拆解 → 测试 → 部署**。

## 写作约定

1. **文件命名**：中文语义命名，复杂版本用 `-v1/-v2` 后缀，例如 `PRD-v1.md`。
2. **引用调研**：写需求/方案时，尽量回引 `analysis/*_需求摘要.md` 作为证据来源，保证「**每条需求都能追溯到某次访谈**」。
3. **改前查 git**：覆盖前先 `git status`，按项目 `git 兜底` 规则操作。
4. **大文件不入库**：原型图、PDF 等大文件放对象存储 / 飞书，文档里贴链接。
