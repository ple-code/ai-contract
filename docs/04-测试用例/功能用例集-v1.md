## 一、AUTH — 登录与岗位

### AUTH-01 正常登录
- **前置**：数据库已种子化（admin 存在、enabled=true）
- **步骤**：访问 `/login` → 输入 `admin / Admin@123` → 点"登录"
- **预期**：跳转 `/`；左侧菜单出现「系统管理」分组；审计日志多一条 `login`

### AUTH-02 错误密码
- **前置**：同上
- **步骤**：输入 `admin / wrong`
- **预期**：页面提示"用户名或密码错误"；不跳转；不写 login 审计

### AUTH-03 账号禁用后登录
- **前置**：管理员在「用户管理」停用某账号
- **步骤**：用该账号登录
- **预期**：提示"账号已禁用"；登录失败

### AUTH-04 首次登录选岗位
- **前置**：新建一个无岗位账号（直接 DB INSERT 绕过 UI）
- **步骤**：该账号登录
- **预期**：弹出「请选择你的岗位」modal；选择并"进入系统"后才能进 `/`

### AUTH-05 登出
- **前置**：已登录
- **步骤**：点头像 → "退出登录"
- **预期**：清除 Cookie；跳转 `/login`；审计多一条 `logout`

### AUTH-06 7 天免登录
- **前置**：登录成功
- **预期**：浏览器 Cookie `access_token` HttpOnly、max-age=7天；7 天内重开浏览器不再登录

## 二、UPL — 合同上传

### UPL-01 上传新合同（docx）
- **前置**：已登录；管理员已配置 AI 网关
- **步骤**：「全部合同」→「＋上传合同」→ 选择一份采购合同 docx → 上传归属选「新合同」→「上传并解析」
- **预期**：跳转 `/contract/:id`；提示"已上传，AI 正在解析合同…"；类型识别为「采购合同」

### UPL-02 不支持的文件类型
- **前置**：同上
- **步骤**：选择 `.txt` 或 `.jpg`
- **预期**：提示"支持 .docx 和 .pdf 格式"；不上传

### UPL-03 不支持的合同类型
- **前置**：同上
- **步骤**：上传一份明显不是采购合同的 docx（如纯劳动合同）
- **预期**：返回 `{ok:false, error:"unsupported_type"}`；前端提示"暂无法识别合同类型"；列出当前支持的类型

### UPL-04 上传已有合同的新版本
- **前置**：库内已有合同 A（v1）
- **步骤**：上传归属选「已有合同的新版本」→ 搜索选中 A → 上传一份修改后的 docx
- **预期**：A 变为 v2；工作台显示基准「v1（上一版本·比对基准）」→「v2 当前」；自动产出 diff

### UPL-05 重复合同识别（编号撞号）
- **前置**：库内已有合同，编号 `CG-2026-001`
- **步骤**：以"新合同"上传另一份编号同为 `CG-2026-001` 的文件
- **预期**：弹框"疑似为已有合同的新版本"，方法标识为编号撞号；三选一按钮可见

### UPL-06 重复合同识别（AI 相似度）
- **前置**：库内已有合同，新文件编号为空但内容高度雷同
- **步骤**：以"新合同"上传
- **预期**：弹框显示"AI 判断与「XX」可能是同一交易的不同版本（相似度 NN%）"；可选关联/仍作新合同/取消

### UPL-07 模板甲方为空 vs 真合同
- **前置**：库内有已签真合同（甲方=具体公司名）
- **步骤**：上传一份甲方空白模板
- **预期**：AI 相似度判定强制返回不相似（match_id=null，score≤30）；不弹框

### UPL-08 上传含表格的 docx
- **前置**：准备含「采购清单」「付款阶段」两个表格的合同
- **步骤**：上传
- **预期**：工作台条款正文渲染为带表头的 HTML 表格（`.md-table`），合并单元格被识别为 colspan

### UPL-09 扫描版 PDF
- **前置**：图片型 PDF
- **步骤**：上传 `.pdf`
- **预期**：系统走 OCR（去红印章），输出可读条款文本

## 三、REV — AI 初审

### REV-01 自动触发初审
- **前置**：上传成功后状态为「AI初审中」，AI 已配置
- **步骤**：进入工作台
- **预期**：自动开始初审；初审中弹框"AI 初审进行中"，确认后只显示左+中两栏

### REV-02 手动触发初审
- **前置**：合同状态「AI初审中」，AI 已配置，未自动触发
- **步骤**：点"开始 AI 初审"
- **预期**：流式展示进度（`审查进度：N / M`）；按条款实时出 finding

### REV-03 AI 未配置
- **前置**：管理员未配置网关
- **步骤**：点"开始 AI 初审"
- **预期**：Toast 错误"尚未配置 AI 模型…"；跳转 `/admin/config`

### REV-04 三立场结果切换
- **前置**：初审已完成
- **步骤**：在顶部「审阅立场」依次点 甲方/乙方/中立
- **预期**：每次切换**不重新调 AI**，瞬时返回对应立场 finding；立场按钮高亮变化

### REV-05 初审后状态推进
- **前置**：初审运行结束
- **预期**：版本状态自动从「AI初审中」→「待人工复核」；出现"完成复核"按钮

### REV-06 每条款含完整字段
- **预期**：finding 含 `risk_level`(high/medium/low)、`finding`、`suggestion`、`legal_basis`（数组）、`stance_note`

### REV-07 法条引用不杜撰
- **前置**：条款命中违约金标签
- **预期**：AI 输出的法律依据都在注入的《民法典》条款范围内（不超出 seed_legal 的条号集合）

## 四、WB — 工作台复核

### WB-01 岗位关注自动定位
- **前置**：用户岗位=财务
- **步骤**：进入一份合同
- **预期**：Toast"已按【财务】岗位定位到关注的条款"；左侧 nav 对应条款带 ★；自动滚动到该条款

### WB-02 接受/拒绝表态
- **前置**：合同有比对基准（非首次上传），状态「待人工复核」
- **步骤**：在某条款点"接受"
- **预期**：条款标绿；左侧 nav 显示「接受」标签；变更记录 + 审计各 +1

### WB-03 撤销表态
- **前置**：WB-02 已接受
- **步骤**：再次点"接受"
- **预期**：取消接受；状态恢复；审计多一条 `undo_decision`

### WB-04 批注
- **步骤**：点"批注" → 输入文字 → 保存
- **预期**：条款下出现"人工批注 · X"；变更记录 +1

### WB-05 一键应用建议
- **前置**：条款有 AI 建议
- **步骤**：点"应用建议"
- **预期**：条款正文被替换为建议文本；按钮变"撤销应用"；保留原正文快照

### WB-06 撤销应用
- **前置**：WB-05 已应用
- **步骤**：点"撤销应用"
- **预期**：还原为原条款文本

### WB-07 立场锁定
- **前置**：条款已有任何决定/批注/应用
- **步骤**：尝试切换立场
- **预期**：Toast 错误"已有复审留痕，立场已锁定"；立场按钮 disabled

### WB-08 法条 hover 浮窗
- **步骤**：鼠标悬停在条款下方的"民法典 第五百八十五条"链接
- **预期**：即时浮窗显示法律名、条号、要点片段（无 1s 延迟）

### WB-09 关键字段变更摘要
- **前置**：v2 相对 v1 修改了合同总价/付款账期
- **预期**：中栏顶部出现"关键字段变更摘要"区，显示 `12,000 元 → 15,000 元` 类字段级变更；按立场显示风险标签（如甲方立场"需核价"）

### WB-10 完成复核
- **前置**：状态「待人工复核」
- **步骤**：点"完成复核" → modal 确认
- **预期**：状态 →「复核完成」；步骤条三步全绿；审计 +1 `finalize`

### WB-11 拖动分隔
- **步骤**：拖动左/右分隔条
- **预期**：实时调整列宽；松手后写入 localStorage，刷新仍保留

### WB-12 nav hover 浮窗
- **步骤**：鼠标悬停左侧条款导航项
- **预期**：固定浮窗显示条款编号/标题/正文前 140 字

## 附录：回归 Golden Set（建议）

每次代码变更后，至少跑通以下端到端链路：

1. **上传链路**：上传含表格的采购合同 → 类型识别成功 → 条款数 > 0 → 表格正确渲染
2. **初审链路**：触发初审 → SSE 流式进度 → 三立场 finding 齐全 → 状态推进到"待人工复核"
3. **复核链路**：接受/拒绝/批注/应用建议 → 变更记录 + 审计 双写
4. **导出链路**：导出 Word/PDF/报告，文件可正常打开且中文不乱码
5. **重复识别**：上传同编号合同 → 弹框正确，三选一分支都可走通
