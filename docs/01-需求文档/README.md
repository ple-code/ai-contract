# 01-需求文档/

## 放什么

- **PRD**（产品需求文档）：按版本迭代，`PRD-v1.md`、`PRD-v2.md`……
- **需求清单**：从调研摘要提炼的功能点清单，标注优先级（P0/P1/P2）
- **用户故事**：`作为<角色>，我想<动作>，以便<价值>`
- **竞品/法规**：招投标相关法规要求、同类产品对标

## 写作要求

- **每条需求必须能追溯到调研**：在条目末尾标注来源，例如
  > 来源：`analysis/20260626-李颖莹-需求摘要.md#法务合同审查痛点`
- **优先级**：P0 = MVP 必做；P1 = 第二批；P2 = 可延后
- **验收标准**：每条 P0/P1 需求附验收标准（Given-When-Then）

## 起步建议

先写 `需求清单.md`：把 `analysis/` 下所有「需求摘要」里的痛点/诉求汇成一张总表，再分类排优先级。
