# 明衡 · AI 合同审阅系统 — 使用手册 v1

> 面向 **终端用户**（法务/销售/商务/财务/管理员）的操作手册。
> 配套：`docs/02-技术方案/功能实现清单-v1.md`（功能全貌）、`docs/04-测试用例/功能用例集-v1.md`（验收用例）。
> 更新日期：2026-06-29。

---

## 一、快速开始

### 1.1 登录系统

1. 浏览器打开系统地址（部署后访问 `http://<服务器IP>`，本机开发为 `http://localhost:5173`）。
2. 输入 **账号** 与 **密码**（账号由管理员统一开通）。
3. 点「登录」。

> 内置管理员：`admin / Admin@123`（仅用于初始化配置，正式使用请由管理员开通个人账号）。

### 1.2 首次进入：选择岗位

首次登录会弹出「请选择你的岗位」对话框：

- **法务**：合同审核责任人，关注风险条款与关键信息。
- **销售**：提交合同，关注交货/标的/价格。
- **商务**：对接客户，关注商务信息与条款回填。
- **财务**：资金把关，关注付款/结算/违约金。

勾选「记住，下次不再选择」可固定岗位（仍可从头像菜单"更换岗位"改回）。岗位决定后续进入合同时 **系统自动定位到哪些条款**。

## 二、合同上传

### 2.1 上传入口

「全部合同」页右上角点「＋ 上传合同」。

### 2.2 选择上传归属

| 选项 | 含义 | 何时选 |
|------|------|--------|
| **新合同** | 库内没有这份合同，建立新合同档案 | 首次入库 |
| **已有合同的新版本** | 该合同已存在，本次是对方修改后的新版本 | 二次/多次回稿 |

选「新版本」时，需在搜索框搜并选中要关联的已有合同，系统会以所选合同的 **当前版本作为比对基准**。

### 2.3 选择文件并上传

- 支持 **Word（.docx）** 与 **PDF（.pdf，文字版）**；扫描件 PDF 自动走 OCR。
- 拖拽到上传区，或点「选择合同文件」。
- 点「上传并解析」。

### 2.4 上传后会发生什么

1. 系统解析合同 → 按条款切分 → 抽取合同编号与甲乙方等摘要。
2. **识别合同类型**：当前仅支持「采购合同」。若无法识别，会提示并给出当前支持的类型列表。
3. **重复合同识别**：
   - 若合同编号与库内某份一致 → 弹框提示"编号撞号"。
   - 若 AI 判断与库内合同高度相似（如同一交易不同版本）→ 弹框显示相似度。
   - 三选一：**关联为新版本** / **仍作为新合同** / **取消**。
4. 上传成功 → 自动跳转工作台，触发 AI 初审。

## 三、AI 初审（自动）

进入工作台后，若 AI 已配置且合同状态为「AI初审中」，**系统会自动开始初审**，无需手动操作。

### 3.1 初审过程

- 顶部出现"AI 正在解析合同并初审…"，并显示进度（如 `审查进度：6 / 36`）。
- 初审进行中若进入详情，会弹窗提示"暂时只能查看合同解析内容"，确认后右栏预览会隐藏，聚焦条款原文。
- 初审完成后：合同状态自动变为「待人工复核」，并 Toast 提示。

### 3.2 三立场初审

系统 **一次性生成 甲方 / 乙方 / 中立 三个立场** 的审查结果，顶部「审阅立场」可切换：

| 立场 | 视角 |
|------|------|
| **甲方**（采购方） | 保护我方利益，违约责任越大越好、按期交货 |
| **乙方**（供方） | 保护供方利益，违约责任越小越好、付款越宽松越好 |
| **中立** | 公平视角，识别明显不平等条款 |

切换立场 **不重新调用 AI**，瞬时切换。

> ⚠️ **立场锁定**：一旦在任一条款做了"接受/拒绝/批注/应用建议"，立场会锁定，无法再切换（避免不同立场的建议与已应用正文冲突）。

### 3.3 AI 未配置时

若管理员尚未配置 AI 模型，点"开始 AI 初审"会提示并跳转到系统配置页。请联系管理员配置后再审。

## 四、人工复核工作台

工作台分三栏：**左侧条款导航 / 中间条款卡片 / 右侧全文预览**。两侧分隔条可拖动调整宽度（自动记忆）。

### 4.1 岗位关注条款自动定位

进入合同后，系统按你的岗位（或你在「我的关注点」里设置的偏好）自动定位到 **第一条关注的条款**，并 Toast 提示"已按【XX】岗位定位…"。左侧导航中关注条款带 **★** 标记。

### 4.2 阅读条款

- 中栏每张条款卡片：左侧为「条款内容」原文，右侧为「AI 解读 / 风险」。
- AI 解读包含：**风险等级**（高/中/低，彩色标签）、**风险说明**、**立场化分析**、**修改建议**、**法律依据**。
- 鼠标悬停在法律依据（如「民法典 第五百八十五条」）会即时显示条文要点浮窗。
- 鼠标悬停左侧导航项会浮窗预览该条款正文前 140 字。

### 4.3 表态：接受 / 拒绝

> 仅在合同 **有版本比对基准**（非首次上传）时可用。

- 点「接受」：条款标记为绿色，左侧导航显示「接受」标签。
- 再点一次「接受」：取消接受。
- 点「拒绝」：同理。

### 4.4 批注

1. 点条款右上「批注」按钮。
2. 输入批注内容（例：*账期偏长，建议与对方协商缩短*）。
3. 点「保存批注」。

批注会显示在条款正文下方，并写入变更记录。

### 4.5 一键应用 / 撤销 AI 建议

- 点「应用建议」：把 AI 给出的修改建议 **直接替换** 条款正文；按钮变「撤销应用」。
- 点「撤销应用」：还原为原条款正文。

### 4.6 关键字段变更摘要（有基准时）

中栏顶部显示「关键字段变更摘要」：合同总价 / 付款账期 / 违约金比例 / 交货期 等核心字段从基准版本到当前版本的 **from → to** 变化，并按当前立场给出风险标签（如甲方立场"需核价"、乙方立场"账期延长·不利"）。

### 4.7 完成复核

1. 所有条款处理完毕后，点右上「完成复核」。
2. 在弹窗中点「复核完成」。
3. 合同状态变为「复核完成」，顶部步骤条三步全绿。

## 五、合同列表与管理

### 5.1 搜索与筛选

- 搜索框：按合同 **名称 / 编号** 模糊搜索。
- 「全部状态」下拉：单选筛选 **AI初审中 / 待人工复核 / 复核完成**。

### 5.2 进入合同

点击任意行进入该合同工作台。

### 5.3 行操作菜单

每行右侧「操作 ▾」展开：

- **变更记录**：弹窗显示该合同所有版本的事件流（上传/比对/接受/批注/应用…），按版本分组，可下载每个版本的原始上传文件。
- **下载原文件**：下载用户最初上传的 docx/pdf（非修订版）。
- **导出** → 三级菜单：
  - **Word**（.docx）：基于模板渲染的修订稿
  - **PDF**（.pdf）：修订稿的 PDF 版
  - **审查报告**（.docx）：结构化审查报告

### 5.4 编号缺失提示

若原始文件中没有合同编号，列表「编号」列显示「?」，鼠标悬停提示"原始文件中没有合同编号"。

## 六、导出

| 类型 | 内容 | 适用场景 |
|------|------|----------|
| **Word 修订稿** | 基于采购合同模板渲染，含已应用建议后的条款正文 | 发给对方签字 |
| **PDF 修订稿** | Word 修订稿的 PDF 版 | 不可编辑场景 |
| **审查报告** | 合同信息表 + 风险汇总（高/中/低计数）+ 逐条 AI 发现 + 人工决定与批注 + 法律依据 | 内部存档、汇报 |

工作台右上角「导出 ▾」也可触发同样的导出。

## 七、我的关注点

每个人可以在岗位默认基础上 **微调关注点**（只影响自己）：

1. 点左下头像 → 「我的关注点」。
2. 勾选/取消 12 类条款标签（价格/付款/违约金/交付/质保/保密/知识产权/合同效力/解除终止/争议解决/不可抗力/数据安全）。
3. 「默认」标记来自岗位配置；可取消或额外勾选。
4. 点「保存」。或点「恢复岗位默认」回到岗位默认。

**优先级**：个人关注点 > 管理员岗位配置 > 系统默认。

## 八、知识库（只读）

### 8.1 法律法规

左侧菜单「法律法规」：浏览 AI 审查所依据的法律条文（民法典合同编、数据安全法、个人信息保护法等）。

- 按法律名 **分组折叠** 展示。
- 顶部搜索条号/内容/关键词；按法律名过滤。
- 每条条文带 **适用标签**（彩色），与条款 type_tags 对应。

### 8.2 审查规则

左侧菜单「审查规则」（或 `/legal?tab=rule`）：查看内置的 8 条确定性审查规则。

- 规则名 / 适用类型 / 触发条件与命中关键词 / 风险等级 / 处理建议。
- 按风险等级过滤；关键词搜索。

> MVP 阶段知识库为 **只读**，由系统初始化注入。如需补充，请联系研发修改种子文件。

## 九、管理员操作

> 仅 **管理员** 角色可见左侧「系统管理」分组。

### 9.1 用户管理

- 「＋新增用户」：填写用户名、姓名、岗位、角色、初始密码（默认 `Init@123`）。
- 启用/停用：停用后该用户立即无法登录。
- 搜索用户名/姓名。

### 9.2 系统配置

5 个 Tab：

#### ① 模型配置（必填）
- **网关地址**：OpenAI 或 Anthropic 兼容端点（如智谱 CodePlan 用 `https://open.bigmodel.cn/api/anthropic`）。
- **API Token**：留空表示不修改（已配置显示 ••••••）。
- **默认模型 / 敏感模型**：模型名（如 `glm-4.5-air`）。
- 「连通测试」：发一条测试调用验证配置是否可用。
- 「保存」。

> 协议（openai/anthropic）由后端环境变量 `AI_PROTOCOL` 决定，前端无需关心。

#### ② 岗位关注点
4 岗位 × 12 标签的复选矩阵；勾选后保存，影响所有用户进入合同时的默认定位。

#### ③ 安全与存储（只读）
展示默认安全基线：TLS 全链路、磁盘+文件加密、敏感合同强制本地推理、账号分配制 + RBAC、全量审计。

#### ④ 数据留存（只读）
展示合同/审计日志留存策略默认值。

#### ⑤ 关于
版本信息、部署形态、默认推理位置。

### 9.3 审计日志

- 按 **操作类型** 过滤（登录/上传/AI 初审/复审决定/批注/完成复核/导出…）。
- 分页浏览，每页 30 条。
- 记录：时间、用户、岗位、操作、对象、IP。

## 十、常见问题

| 现象 | 原因 / 解决 |
|------|-------------|
| 上传提示"暂不支持 .xxx 格式" | 当前仅支持 `.docx` 和 `.pdf`；转换格式后再传 |
| 上传提示"暂无法识别合同类型" | 内容不像采购合同；MVP 仅支持采购合同 |
| 点"开始 AI 初审"提示未配置 | 管理员需先在「系统配置 → 模型配置」填网关与 Token |
| 初审长时间不动 | 合同条款多时初审较慢（每条款 ×3 立场）；查看审计日志确认是否在跑 |
| 切换立场被拒 | 本版本已有复审留痕，立场已锁定，无法切换 |
| 接受/拒绝按钮不可点 | 仅在合同 **有版本比对基准**（非首次上传）时可用，首次上传仅可批注 |
| 导出 PDF 失败 | 服务器需安装 LibreOffice；联系管理员检查容器内 `libreoffice` 命令 |
| 登录提示"账号已禁用" | 联系管理员启用账号 |
| 编号列显示「?」 | 原始文件里没有合同编号字段，属正常 |

## 十一、最佳实践

1. **统一命名**：上传的文件名建议含「合同名 + 版本 + 日期」，便于追溯（系统也会保留原文件名）。
2. **优先用新版本模式**：对方修改后回稿时，用「已有合同的新版本」上传，可自动 diff 并定位变更。
3. **善用岗位定位**：财务重点关注付款/价格，法务重点关注违约/解除/争议解决；先看关注条款再看全文。
4. **复核留痕**：所有接受/拒绝/批注都进变更记录与审计，团队可追溯，请规范填写批注理由。
5. **完成复核前**：确认所有关注条款已处理；完成后无法再切换立场。
